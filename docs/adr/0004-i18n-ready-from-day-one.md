# מחרוזות UI עוברות דרך i18next מ-day 1, גם כשיש רק עברית

**Status:** accepted

המסמך קובע שהאתר "בשלב הראשון יפותח בעברית אך יש לפתח תמיכה בריבוי שפות לעתיד (רוסית/צרפתית/אנגלית)". כל מחרוזת UI תיכתב מההתחלה דרך i18next (מילון `he.json` בלבד כרגע), לא hardcoded inline בקומפוננטות — גם אם בפועל שפה אחת זמינה היום.

**הנימוק:** זו החלטה "קל עכשיו / יקר אחר כך" קלאסית. העלות השולית של לעטוף מחרוזת ב-`t('key')` מהיום הראשון אפסית; refactor של כל הקומפוננטות בעתיד כדי לחלץ מחרוזות hardcoded הוא עבודה משמעותית וגם מקור-בא לבאגים (פירוט RTL/LTR לפי שפה כבר רלוונטי גם היום בעברית מול אנגלית טכנית בממשק).

**מה זה לא:** אין צורך לתרגם בפועל לרוסית/צרפתית/אנגלית בסבב הזה — רק לא לחסום את האפשרות.
