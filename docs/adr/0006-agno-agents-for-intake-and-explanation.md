# שני סוכני AI (Agno/Python) — קבלה חלופית לטופס, והסבר אישי על המשכנתא

**Status:** accepted

מוסיפים שני פיצ'רי **סוכן** (ראו CONTEXT.md) שלא היו במסמך הלקוח המקורי — בידול-מוצר שהצוות בחר:

1. **סוכן-קבלה** — **אופציה חלופית** לטופס המדורג הסטטי של שאלון "משכנתא חדשה" (ARCHITECTURE.md §3). המשתמש בוחר **טופס או סוכן** — שניהם קיימים במקביל, לא מחליפים זה את זה. הסוכן חייב לאסוף את **כל** השדות הנדרשים בדיוק כמו הטופס (כולל תנאים לפי סוג הלוואה/מספר לווים). השלמות (אילו שדות עדיין חסרים) מנוהלת ע"י tool דטרמיניסטי בקוד — לא שיקול-דעת LLM — כך שכללי-החובה לא יכולים "להישחק" בפרומפט. פלט הסוכן הוא אותו מבנה דאטה (`לווים`/`financial`) שהטופס מפיק, ונשמר ע"י הקליינט ל-localStorage בדיוק כמו היום (ADR-0001) — לא ערוץ-נתונים נפרד.
2. **סוכן-הסבר** ("הסבר לי את המשכנתא שלי") — RAG על מילון המושגים (נספח א' במסמך הלקוח) בשילוב tool שמביא את הנתונים האמיתיים של המשתמש (קורא לאותו Cloud Function המוסתר של מנוע החישוב, ARCHITECTURE.md §6). מסביר **רק** מה שמעוגן בגלוסר + בנתונים האמיתיים של המשתמש — לא ידע כללי.

**Framework: Agno (Python), לא LangChain.** בריפו הדוגמאות של הקורס (`kerenkalif/DevelopingWithAi-CourseCodeExamples`, תיקייה `06- agents`) Agno הוא ה-framework היחיד שמכיל ביחד RAG (`05_agno_rag.py`), custom tools (`04_agno_custom_tools.py`), זיכרון רב-פעמי (`06_agno_memory.py`), ו-human-in-the-loop (`08_agno_human_in_the_loop.py`) — מתאים ל-2 הפיצ'רים בלי לערבב framework נוסף.

**תוצאה ארכיטקטונית: Cloud Functions בשני runtimes.** מנוע החישוב (מקור JS, ARCHITECTURE.md §6), כפתור-התשלום (ADR-0003) וולידציית הקובץ נשארים Node/TypeScript ללא שינוי. שני פונקציות הסוכנים החדשות יהיו **Python** (Firebase 2nd-gen Cloud Functions תומך ב-runtime נפרד לכל פונקציה). זו תוצאה לא-מובנת-מאליה שחשוב לתעד כדי שלא "יתקנו" אותה בטעות לאחידות runtime.

**Human-in-the-loop כאן ≠ `requires_confirmation` של Agno.** ה-pattern של Agno מיועד לפעולות עם תופעת-לוואי בתוך הסוכן עצמו (כמו שליחת מייל). אצלנו, השמירה בפועל קורית בקליינט (ל-localStorage), לא בתוך הסוכן — אז הפתרון הוא מסך recap/אישור ב-UI אחרי כל שדה שחולץ, לפני שהוא נחשב סופי, לא מנגנון ה-confirmation הפנימי של Agno.

**נשאר פתוח (לא חוסם):** ספק embeddings ל-RAG — דוגמת הקורס משתמשת ב-OpenAI embedder גם כששאר הסוכן הוא Claude; צריך להחליט אם להוסיף תלות ב-OpenAI API key רק בשביל embeddings, או למצוא embedder אחר שAgno תומך בו.
